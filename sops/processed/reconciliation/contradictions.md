## CON-001: Invoice Creation Method

**Severity:** CRITICAL
**SOP:** Public Invoice
**Step:** 3.1
**Source:** `SOPs/raw_sops/public_invoice_sop.md`

### What SOP Says

> At the top of the quote page, click "Convert" > "Convert to Invoice"

### What CRM Actually Does

Invoice is automatically created by workflow when Quote_Stage is changed to "Closed Won".

**Workflow Details:**
- **Name:** Update Deal and Invoice on Quote Closed Won
- **ID:** 52330000002460325
- **URL:** https://crm.zoho.com.au/crm/org7003757385/settings/workflow-rules/52330000002460325
- **Trigger:** Quote_Stage field update to "Closed Won"
- **Actions:**
  - Creates Invoice record from Quote
  - Updates Deal stage to Closed Won
  - Sets Quote_Won_Date

### Impact

| Impact Type | Description |
|-------------|-------------|
| **Duplicate Records** | User creates manual invoice, then workflow creates another |
| **Missing Field Updates** | Manual conversion bypasses Quote_Won_Date, Deal stage update |
| **Reporting Errors** | Invoice count inflated, revenue potentially double-counted |
| **Audit Trail Gap** | No tracking of quote acceptance date |

### Root Cause

SOP was written before automation was implemented, or based on manual process that has since been automated.

### Resolution

**Immediate:** Update SOP Step 3 to read:

```markdown
### Step 3: Mark Quote as Accepted

- In the Quote record, change Quote_Stage to "Closed Won"
- Save the record
- The system will automatically:
  - Create an Invoice record
  - Update the Deal stage to Closed Won
  - Set the Quote_Won_Date
- Navigate to Invoices module to find the new invoice
- Do NOT use the "Convert" button - this bypasses tracking
```

## CON-002: Invoice Send Method

**Severity:** HIGH
**SOP:** Public Invoice
**Step:** 4.1
**Source:** `SOPs/raw_sops/public_invoice_sop.md`

### What SOP Says

> Click on "Send Email" from within the invoice page.
> Choose the "Public Invoice Template"
> Click "Next" and "Send"

### What CRM Actually Does

Invoice is sent via workflow when Status field is changed to "Sent".

**Workflow Details:**
- **Name:** Send Invoice
- **ID:** 52330000004534311
- **URL:** https://crm.zoho.com.au/crm/org7003757385/settings/workflow-rules/52330000004534311
- **Trigger:** Status field update to "Sent"
- **Actions:**
  - Sends invoice email using configured template
  - Sets Invoice_Sent_Date

### Impact

| Impact Type | Description |
|-------------|-------------|
| **Workflow Bypass** | Clicking button may not trigger workflow |
| **Missing Tracking** | Invoice_Sent_Date not populated |
| **Integration Issues** | Xero sync and other integrations may not fire |
| **Template Mismatch** | User cannot select template when using correct method |

### Root Cause

Two methods exist for sending invoices, but only the Status field method triggers proper tracking.

### Resolution

**Immediate:** Update SOP Step 4 to read:

```markdown
### Step 4: Send Invoice

- Change the Status field to "Sent"
- Save the record
- The system will automatically:
  - Send the invoice email
  - Set Invoice_Sent_Date
  - Trigger Xero sync (if applicable)
- Note: Template is configured in workflow, not selected per invoice
- Do NOT use the "Send Email" button - this bypasses tracking
```

## CON-003: Deal Naming Convention (Pipeline SOP)

**Severity:** HIGH
**SOP:** Deal Pipeline Management
**Step:** 2.2
**Source:** `SOPs/raw_sops/deal_pipeline_management.md`

### What SOP Implies

User should manually enter Deal Name as a mandatory field.

> Mandatory fields to populate:
> - Deal Name

### What CRM Actually Does

Deal Name is auto-generated by workflow on record creation.

**Workflow Details:**
- **Name:** Deal Naming Convention
- **ID:** 52330000002967279
- **URL:** https://crm.zoho.com.au/crm/org7003757385/settings/workflow-rules/52330000002967279
- **Trigger:** Record create
- **Format:** `P - [Company] - [Location] - [Course]`

### Impact

| Impact Type | Description |
|-------------|-------------|
| **Wasted Effort** | User enters name that is immediately overwritten |
| **User Confusion** | Manual entry disappears on save |
| **Inconsistent Naming** | If workflow fails, manual name persists |

### Root Cause

SOP documents manual process, but automation has been implemented.

### Resolution

**Immediate:** Update SOP Step 2 to read:

```markdown
### Step 2: Creating Deals

- When creating a Deal, set:
  - Contact_Name (student)
  - Account_Name (company/payer)
  - Courseaa (course lookup)
  - Course_Start_Date
- Do NOT manually enter Deal Name
- The system will auto-generate using format: P - [Company] - [Location] - [Course]
- Verify the auto-generated name after save
```

## CON-004: Stage Name "New"

**Severity:** HIGH
**SOP:** Deal Pipeline Management
**Step:** 3.1
**Source:** `SOPs/raw_sops/deal_pipeline_management.md`

### What SOP Says

> Move Sub Deals through the pipeline stages as they progress:
> - New

### What CRM Actually Has

Stage "New" does not exist. Initial stage is "Qualification".

**Actual CRM Stages:**
1. Qualification
2. Negotiation/Review
3. Ready to Quote
4. Awaiting ZipPay Confirmation
5. Awaiting Purchase Order
6. Awaiting Invoice Payment
7. Unused
8. Closed Won
9. Closed Lost

### Impact

| Impact Type | Description |
|-------------|-------------|
| **Stage Not Found** | User cannot find "New" in dropdown |
| **Process Confusion** | User may skip to wrong stage |
| **Reporting Gap** | Deals in initial stage not tracked as expected |

### Root Cause

SOP uses conceptual stage names that differ from CRM configuration.

### Resolution

**Immediate:** Update SOP to use actual CRM stage names:

```markdown
### Step 3: Updating Deal Stages

Move Deals through the pipeline stages:
- **Qualification** (initial stage) - Verify deal details
- **Negotiation/Review** - Chase requirements, send quote
- **Ready to Quote** - Quote auto-generates
- **Awaiting ZipPay Confirmation** (if applicable)
- **Awaiting Purchase Order** - Chase PO
- **Awaiting Invoice Payment** - Chase payment
- **Closed Won** - Via Quote acceptance or PO entry
- **Closed Lost** - Enter Loss_Reason
```

## CON-005: Stage Name "Quoted"

**Severity:** MEDIUM
**SOP:** Deal Pipeline Management
**Step:** 3.3
**Source:** `SOPs/raw_sops/deal_pipeline_management.md`

### What SOP Says

> Move Sub Deals through the pipeline stages:
> - Quoted

### What CRM Actually Has

Stage "Quoted" does not exist. After Ready to Quote, Deals return to "Negotiation/Review".

### Impact

| Impact Type | Description |
|-------------|-------------|
| **Stage Not Found** | User cannot find "Quoted" in dropdown |
| **Missing Tracking** | No stage to indicate quote has been sent |
| **Reporting Gap** | Cannot report on deals with pending quotes |

### Root Cause

SOP assumes stage that was never created in CRM, or was removed.

### Resolution

**Option A - Update SOP:**

```markdown
After sending quote, return Deal to "Negotiation/Review" stage.
Track quote status in Quotes module via Quote_Stage field:
- Draft
- Sent
- Closed Won
- Declined
```

**Option B - Add CRM Stage:**

Consider adding "Quote Sent" stage if business requires tracking at Deal level.
